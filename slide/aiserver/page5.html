핵심 패키지 역할

controller

REST API 요청 처리 및 WebSocket 메시지 라우팅 담당

service

비즈니스 로직의 핵심. SSH 연결, AI API 호출, 트랜잭션 관리

repository

JPA를 이용한 DB 접근 계층. CRUD 작업 추상화

entity

DB 테이블과 매핑되는 도메인 객체 정의

설계 원칙 (Design Principles)

  • 단방향 의존성: Controller → Service → Repository 순서의 흐름 엄수
  • 관심사 분리: SSH 통신과 AI 로직은 별도 서비스로 완전히 분리하여 유지보수성 향상
  • 최소 권한 엔티티: 도메인 모델은 필요한 필드만 정의하여 가볍게 시작하고 점진적으로 확장
